使用 namedtuple 写出简练的代码

  2021-12-06   Python   python collections 字数统计: 4460字 阅读时长: 9分
  • 原文 : https://realpy.thoncom/python-namedtuple/

1. 使用 namedtuple 编写 Pythonic 代码

  • Pythonnamedtuple()是工厂函数，可在中使用collections。它允许您tuple使用命名字段创建子类。您可以使用点符号和字段名称（如中的）访问给定命名元组中的值obj.attr。
  • namedtuple创建Python的目的是通过提供一种使用描述性字段名而不是整数索引来访问值的方法来提高代码的可读性，而在大多数情况下，它们不提供有关值是什么的任何上下文。此功能还使代码更整洁，更易于维护。
  • 相反，在常规元组中使用对值的索引可能很烦人，难以阅读且容易出错。如果元组有很多字段并且构造得离您使用它的地方很远，则尤其如此。
  • 除了命名元组的主要功能外，您还将发现它们：
    • 是不可变的数据结构
    • 具有一致的哈希值
    • 可以用作字典键
    • 可以成套存放
    • 根据类型和字段名称使用有用的文档字符串
    • 提供有用的字符串表示形式，以一种name=value格式打印元组内容
    • 支持索引
    • 提供额外的方法和属性，例如._make()，_asdict()，._fields，等
    • 是向后兼容与普通的元组
    • 与常规元组具有相似的内存消耗
  • 通常，您可以namedtuple在需要类似元组的对象的任何地方使用实例。命名元组的优点是它们提供了一种使用字段名称和点表示法访问其值的方法。这将使您的代码更加Pythonic。
  • 通过此简介，您可以更深入地研究如何在代码中创建和使用它们。

2. 使用 namedtuple() 创建元组类

  • 您可以使用namedtuple()创建带有字段名称的不变且类似元组的数据结构。

  • 您可能希望使用不可变的数据结构来表示给定点。使用常规元组创建二维点的方法如下：

    >>> # Create a 2D point as a tuple
>>> point = (2, 4)
>>> point
(2, 4)

>>> # Access coordinate x
>>> point[0]
2
>>> # Access coordinate y
>>> point[1]
4

>>> # Try to update a coordinate value
>>> point[0] = 3
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

  • 您可以point使用Regular创建不可变的二维tuple。这段代码有效：您有一个point具有两个坐标的，并且您无法修改任何这些坐标。但是，此代码可读吗？您能先告诉一下0和1索引是什么意思吗？为了避免这些歧义，您可以使用namedtuple如下所示的代码：

    >>> from collections import namedtuple

>>> Person = namedtuple("Person", "name children")
>>> john = Person("John Doe", ["Timmy", "Jimmy"])
>>> john6
Person(name='John Doe', children=['Timmy', 'Jimmy'])
>>> id(john.children)
139695902374144

>>> john.children.append("Tina")
>>> john
Person(name='John Doe', children=['Timmy', 'Jimmy', 'Tina'])
>>> id(john.children)
139695902374144

2.1. 提供必要的参数

  1. typename提供由namedtuple返回的类名namedtuple()。您需要将带有有效Python标识符的字符串传递给此参数。

  2. field_names提供您将用于访问元组中的值的字段名称。您可以使用以下方式提供字段名称：

    • 字符串的迭代，例如[“field1”, “field2”, …, “fieldN”]
    • 一个字符串，每个字段名用空格分隔，例如 “field1 field2 … fieldN”
    • 一个字符串，每个字段名称用逗号分隔，例如 “field1, field2, …, fieldN”
    >>> from collections import namedtuple

>>> # A list of strings for the field names
>>> Point = namedtuple("Point", ["x", "y"])
>>> Point
<class '__main__.Point'>
>>> Point(2, 4)
Point(x=2, y=4)

>>> # A string with comma-separated field names
>>> Point = namedtuple("Point", "x, y")
>>> Point
<class '__main__.Point'>
>>> Point(4, 8)
Point(x=4, y=8)

>>> # A generator expression for the field names
>>> Point = namedtuple("Point", (field for field in "xy"))
>>> Point
<class '__main__.Point'>
>>> Point(8, 16)
Point(x=8, y=16)

  3. 可以使用任何有效的Python标识符用于字段名称，除：

    • 名称以下划线（_）开头
    • Python keywords

如果您提供的字段名称违反了上述任一条件，那么您将获得ValueError

2.2. 使用可选参数

  1. rename

    • 如果设置rename为True，则所有无效的字段名称都会自动替换为位置名称。

  2. defaults

    • 此参数默认为None，这意味着这些字段将没有默认值。您可以将defaults值设置为可迭代的。在这种情况下，namedtuple()将defaultsiterable中的值分配给最右边的字段
    from collections import namedtuple

Developer = namedtuple(
    "Developer",
    "name level language",
    defaults=["Junior", "Python"]
)

Developer("John")

  3. module

    • 如果为该参数提供有效的模块名称，则结果的.__module__属性将namedtuple设置为该值。此属性保存在其中定义了给定函数或可调用函数的模块的名称：
    >>> from collections import namedtuple

>>> Point = namedtuple("Point", "x y", module="custom")
>>> Point
<class 'custom.Point'>
>>> Point.__module__
'custom'

3. 探索namedtuple类的其他功能

  • 除了从 tuple 继承的 .count(), .index() 等方法外，namedtuple类还提供三种额外的方法和两个属性。

3.1. ._make() : 从可迭代对象创建实例

  1. 可以使用._make()创建命名元组实例。该方法采用一个可迭代的值，并返回一个新的命名元组.

    >>> from collections import namedtuple

>>> Person = namedtuple("Person", "name age height")
>>> Person._make(["Jane", 25, 1.75])
Person(name='Jane', age=25, height=1.75)
  • 首先Person使用创建一个类namedtuple()。然后，您调用._make()中的每个字段的值列表namedtuple。请注意，这._make()是一个类方法，用作替代类的构造函数，并返回一个新的命名元组实例。

3.2. ._asdict() :将namedtuple实例转换成字典

  1. 您可以使用._asdict()将已命名的元组实例转换成字典。此方法返回一个新的字典，该字典使用字段名称作为键。结果字典的键与原始字典中的字段的顺序相同namedtuple

    >>> from collections import namedtuple

>>> Person = namedtuple("Person", "name age height")
>>> jane = Person("Jane", 25, 1.75)
>>> jane._asdict()
OrderedDict([('name', 'Jane'), ('age', 25), ('height', 1.75)])

_asdict() 在 3.8 版更改: 返回一个常规 dict 而不是 OrderedDict。 因为自 Python 3.7 起，常规字典已经保证有序。 如果需要 OrderedDict 的额外特性，推荐的解决方案是将结果转换为需要的类型: OrderedDict(nt._asdict())。

3.3. _replace() : 替换现有namedtuple实例中的字段

  1. 此方法采用形式的关键字参数，field=value并返回一个新namedtuple实例，以更新所选字段的值

    >>> from collections import namedtuple

>>> Person = namedtuple("Person", "name age height")
>>> jane = Person("Jane", 25, 1.75)

>>> # After Jane's birthday
>>> jane = jane._replace(age=26)
>>> jane
Person(name='Jane', age=26, height=1.75)

3.4. 探索其他namedtuple属性

  1. 命名元组还具有两个附加属性：._fields和._field_defaults。第一个属性包含列出字段名称的字符串元组。第二个属性包含一个字典，该字典将字段名称映射到其各自的默认值（如果有）。

  2. 对于._fields，您可以使用它来内省您的namedtuple类和实例。您还可以从现有的类创建新的类:

    >>> from collections import namedtuple

>>> Person = namedtuple("Person", "name age height")

>>> ExtendedPerson = namedtuple(
...     "ExtendedPerson",
...     [*Person._fields, "weight"]
... )

>>> jane = ExtendedPerson("Jane", 26, 1.75, 67)
>>> jane
ExtendedPerson(name='Jane', age=26, height=1.75, weight=67)
>>> jane.weight
67

  3. 您还可以使用Python的._fields遍历给定namedtuple实例中的字段和值zip():

    >>> from collections import namedtuple

>>> Person = namedtuple("Person", "name age height weight")
>>> jane = Person("Jane", 26, 1.75, 67)
>>> for field, value in zip(jane._fields, jane):
...     print(field, "->", value)
...
name -> Jane
age -> 26
height -> 1.75
weight -> 67

  4. 通过快速查询._field_defaults，您可以找出给定字段中的哪些字段namedtuple提供默认值。

    >>> from collections import namedtuple

>>> Person = namedtuple(
...     "Person",
...     "name age height weight country",
...     defaults=["Canada"]
... )

>>> Person._field_defaults
{'country': 'Canada'}

4. 用 namedtuple 编写 Python 代码

4.1. 使用字段名称代替索引

  1. 假设您正在创建绘画应用程序，并且需要根据用户的选择定义要使用的笔属性。您已将笔的属性编码为一个元组：

    pen = (2, "Solid", True)
if pen[0] == 2 and pen[1] == "Solid" and pen[2]:
    print("Standard pen selected")

  • 这行代码定义了具有三个值的元组。您能说出每个值的含义是什么吗？也许您可以猜测第二个值与线条样式有关，但是 2 和 True 的含义是什么？

  • 您可以添加一个不错的注释来为提供一些上下文pen，在这种情况下，您将得到如下所示的结果：

    >>> # Tuple containing: line weight, line style, and beveled edges
>>> pen = (2, "Solid", True)

  • 但是，如果您或其他程序员在别的位置使用 pen 怎么办？他们必须回到该定义处。

  • 替代实现的pen使用namedtuple：

    from collections import namedtuple

Pen = namedtuple("Pen", "width style beveled")
pen = Pen(2, "Solid", True)

if pen.width == 2 and pen.style == "Solid" and pen.beveled:
    print("Standard pen selected")

4.2. 从函数返回多个命名值

  1. 这是一个函数，用于使用namedtuple来阐明divmod()返回的每个数字的含义：

    >>> from collections import namedtuple

>>> def custom_divmod(a, b):
...     DivMod = namedtuple("DivMod", "quotient remainder")
...     return DivMod(*divmod(a, b))
...

>>> custom_divmod(8, 4)
DivMod(quotient=2, remainder=0)

4.3. 减少函数的参数数量

  1. 假设您正在编码用于管理客户信息的应用程序。该应用程序使用数据库来存储客户的数据。为了处理数据和更新数据库，您已经创建了几个函数。您的高级功能之一是create_user() :

    def create_user(db, username, client_name, plan):
    db.add_user(username)
    db.complete_user_profile(username, client_name, plan)

  • 该函数有四个参数。第一个参数db代表您正在使用的数据库。其余参数与给定的客户密切相关。这是减少create_user()使用命名元组的参数数量的好机会：

    User = namedtuple("User", "username client_name plan")
user = User("john", "John Doe", "Premium")

def create_user(db, user):
    db.add_user(user.username)
    db.complete_user_profile(
        user.username,
        user.client_name,
        user.plan
    )
    • 现在create_user()仅接受两个参数：db和user。

4.4. 从文件和数据库读取表格数据

  1. 正在考虑使用Python的csv模块及其模块DictReader来处理文件，但是您还有其他要求-您需要将数据存储到一个不变的轻量级数据结构中。在这种情况下，anamedtuple可能是一个不错的选择：

    name,job,email
"Linda","Technical Lead","linda@example.com"
"Joe","Senior Web Developer","joe@example.com"
"Lara","Project Manager","lara@example.com"
"David","Data Analyst","david@example.com"
"Jane","Senior Python Developer","jane@example.com"

  2. 将数据存储到一个不变的轻量级数据结构中

    import csv
from collections import namedtuple

with open("employees.csv", "r") as csv_file:
    reader = csv.reader(csv_file)
    Employee = namedtuple("Employee", next(reader), rename=True)
    for row in reader:
        employee = Employee(*row)
        print(employee.name, employee.job, employee.email)

5. namedtuple 与其他数据结构比较

5.1. namedtuple vs 字典

  1. 字典是可变的，命名元组是不可变的。这意味着您可以就地修改字典，但不能修改命名元组

  2. 关于内存使用

    # namedtuple_dict_memory.py

from collections import namedtuple
from pympler import asizeof

Point = namedtuple("Point", "x y z")
point = Point(1, 2, 3)

namedtuple_size = asizeof.asizeof(point)
dict_size = asizeof.asizeof(point._asdict())
gain = 100 - namedtuple_size / dict_size * 100

print(f"namedtuple: {namedtuple_size} bytes ({gain:.2f}% smaller)")
print(f"dict:       {dict_size} bytes")

5.2. namedtuple 与数据类

  • 数据类可以被认为是“带有默认值的可变命名元组”。

  • 准确地说，数据类就像是具有类型提示的可变命名元组。“默认值”部分根本没有区别，因为命名元组的字段也可以具有默认值。因此，乍一看，主要区别在于可变性和类型提示。

  • 要创建数据类，您需要从中导入dataclass()装饰器dataclasses。然后，您可以使用常规的类定义语法来定义数据类：

    >>> from dataclasses import dataclass

>>> @dataclass
... class Person:
...     name: str
...     age: int
...     height: float
...     weight: float
...     country: str = "Canada"
>>> jane = Person("Jane", 25, 1.75, 67)
>>> jane
Person(name='Jane', age=25, height=1.75, weight=67, country='Canada')
>>> jane.name
'Jane'
>>> jane.name = "Jane Doe"
>>> jane.name
'Jane Doe'

  • 可以将dataclass()装饰器的frozen参数设置为True并使它们不可变：

    from dataclasses import dataclass

@dataclass(frozen=True)
class Person:
    name: str
    age: int
    height: float
    weight: float
    country: str = "Canada"

jane = Person("Jane", 25, 1.75, 67)
jane.name = "Jane Doe"

  • 数据类后者是不可迭代的.
    解决: 可以添加一个.__iter__()特殊的方法

    from dataclasses import astuple, dataclass

@dataclass
class Person:
    name: str
    age: int
    height: float
    weight: float
    country: str = "Canada"
    def __iter__(self):
        return iter(astuple(self))

for field in Person("Jane", 25, 1.75, 67):
    print(field)

  • 关于内存消耗，命名元组比数据类更轻便。

  • 在属性访问操作方面，两种数据结构都具有相同的性能。

5.3. namedtuple 与 typing.NamedTuple

  • Python 3.5引入了一个称为临时模块的临时模块，typing以支持函数类型注释或类型提示。此模块提供NamedTuple，它是的类型化版本namedtuple。使用NamedTuple，您可以创建namedtuple带有类型提示的类。在Person示例之后，您可以创建一个等效的类型化的命名为tuple，如下所示：

    from typing import NamedTuple

class Person(NamedTuple):
    name: str
    age: int
    height: float
    weight: float
    country: str = "Canada"

issubclass(Person, tuple)

jane = Person("Jane", 25, 1.75, 67)
jane.name

jane.name = "Jane Doe"

  • 使用NamedTuple，您可以创建支持类型提示和通过点表示法进行属性访问的元组子类。由于结果类是元组子类，因此它也是不可变的。

6. 子namedtuple类分类

  • 由于namedtuple类是常规的Python类，因此如果需要提供其他功能，文档字符串，用户友好的字符串表示形式等，则可以将其子类化.

  • 例如，将一个人的年龄存储在一个对象中并不是最佳实践。因此，您可能需要存储出生日期并在需要时计算年龄：

    >>> from collections import namedtuple
>>> from datetime import date

>>> BasePerson = namedtuple(
...     "BasePerson",
...     "name birthdate country",
...     defaults=["Canada"]
... )

>>> class Person(BasePerson):
...     """A namedtuple subclass to hold a person's data."""
...     __slots__ = ()
...     def __repr__(self):
...         return f"Name: {self.name}, age: {self.age} years old."
...     @property
...     def age(self):
...         return (date.today() - self.birthdate).days // 365
...

>>> Person.__doc__
"A namedtuple subclass to hold a person's data."

>>> jane = Person("Jane", date(1996, 3, 5))
>>> jane.age
25
>>> jane
Name: Jane, age: 25 years old.

7. 测量创建时间：tuple vs namedtuple

  • 一个脚本，用于测量动态创建多个元组和命名元组所需的平均时间：

    # tuple_namedtuple_time.py

from collections import namedtuple
from time import perf_counter

def average_time(test_func):
    time_measurements = []
    for _ in range(1_000):
        start = perf_counter()
        test_func()
        end = perf_counter()
        time_measurements.append(end - start)
    return sum(time_measurements) / len(time_measurements) * int(1e9)

def time_tuple():
    tuple([1] * 1000)

fields = [f"a{n}" for n in range(1000)]
TestNamedTuple = namedtuple("TestNamedTuple", fields)

def time_namedtuple():
    TestNamedTuple(*([1] * 1000))

namedtuple_time = average_time(time_namedtuple)
tuple_time = average_time(time_tuple)
gain = namedtuple_time / tuple_time

print(f"tuple:      {tuple_time:.2f} ns ({gain:.2f}x faster)")
print(f"namedtuple: {namedtuple_time:.2f} ns")

  • tuple动态创建对象比创建类似的命名元组要快得多。在某些情况下（例如，使用大型数据库），创建命名元组所需的额外时间会严重影响应用程序的性能，因此，如果您的代码动态创建大量元组，请注意这一点。
