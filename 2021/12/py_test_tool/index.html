# Python 常用的测试框架

  2021-12-06   Python   python pytest 字数统计: 1907字 阅读时长: 4分

单元测试与模块测试

  1. 单元测试
    • 是应用程序中的最小可测试部分
    • 在面向过程中，单元可以是整个模块，但常见的是单个函数或过程
    • 在面向对象中，单元通常是整个接口，例如类，但也可以是单独的方法
    • 单元测试多数情况下有程序员自己完成
  2. 模块测试
    • 模块测试是针对具有明显功能特征的代码块进行的测试
    • 单元测试可能只涉及一小部分的功能
  3. 程序接口
    • 提供了输入输出的类,方法或函数
  4. 协议接口
    • 对地层代码进行了封装，通过协议对外提供调用
  5. 意义
    • 单元测试和接口测试是更早介入测试的两个方面

doctest

功能

  1. doctest 模块会寻找类似于 Python 交互式代码(linux 中输入 python3 后显示的交互窗口)的文本, 然后代替交互窗口执行这些命令, 并判断是否与预期相同.

使用方法

  1. 方法1

    • 创建一个 test.py
    # test.py
def my_function(a, b):
    """
    >>> my_function(2, 3)
    6
    >>> my_function('a', 3)
    'aaa'
    """
    return a * b
    • 执行 > python3 -m doctest test.py (如果测试全通过不会有任何显示)
    • 或 > python3 -m doctest -v test.py (-v 表示 verbose, 总是显示测试结果)
    • 输出
    Trying:
    my_function(2, 3)
Expecting:
    6
ok

    my_function('a', 3)

    'aaa'
ok

    __main__

2 tests in 2 items.
2 passed and 0 failed.
Test passed.

  2. 方法2 : 导入 doctest

    • 创建一个 test2.py
    # test.py
def my_function(a, b):
    """
    >>> my_function(2, 3)
    6
    >>> my_function('a', 3)
    'aaa'
    """
    return a * b

if __name__ == "__main__":
    import doctest
    doctest.testmod()
    • 执行 > python3 test2.py -v
    • 输出同上

  3. 方法3 : 单独创建测试用例

    • 创建一个 test3.py
    def my_function(a, b):
    return a * b

if __name__ == "__main__":
    import doctest
    doctest.testfile("example.txt")
    • 创建一个 example.txt

!> 注意事项

  1. 关于 ‘' 反斜杠

    • 使用 r 来保留 ‘'
    >>> def f(x):

    >>> def f(x):
...     '''Backslashes in a raw docstring: m\\n'''

unittest

功能

  1. 测试用例
    • 一个测试用例是一个独立的测试单元。它检查输入特定的数据时的响应。 unittest 提供一个基类： TestCase ，用于新建测试用例
  2. 不同分类
    • 单元测试 : unittest
    • HTTP 接口自动化 : unittest + Request
    • Web UI 自动化 : unittest + Selenium
    • 移动自动化 : unittest + Appium

基本实例

  1. 测试 3 种字符串

    # test4.py
import unittest

class TestStringMethods(unittest.TestCase):

    def test_upper(self):
        self.assertEqual('foo'.upper(), 'FOO')

    def test_isupper(self):
        self.assertTrue('FOO'.isupper())
        self.assertFalse('Foo'.isupper())

    def test_split(self):
        s = 'hello world'
        self.assertEqual(s.split(), ['hello', 'world'])
        # check that s.split fails when the separator is    not a string
        with self.assertRaises(TypeError):
            s.split(2)

if __name__ == '__main__':
    #默认运行当前测试文件中所有测试用例，直接使用 main() 方法
    unittest.main()
    • python test4.py -v
    test_isupper (__main__.TestStringMethods) ... ok
test_split (__main__.TestStringMethods) ... ok
test_upper (__main__.TestStringMethods) ... ok

------------------------------------------------------------    ----------
Ran 3 tests in 0.001s

OK

  2. unittest 模块可以通过命令行运行模块、类和独立测试方法的测试

    • python -m unittest test_module1 test_module2
    • python -m unittest test_module.TestClass
    • python -m unittest test_module.TestClass.test_method

pytest

1. 基础

  1. 安装 pip install -U pytest
  2. 断言 : assert a == b
  3. 运行 : pytest test.py
  4. 详细打印 : pytest -v test.py
  5. 捕获异常 raise
  6. 使用 class 包装成组

2. 操作

  1. python -m pytest 调用时 将当前路径加入 sys.path
  2. 结束代码 : from pytest import ExitCode
    • 0 : 所有 tests 被收集并成功通过
    • 1 : tests 被收集和运行，但一些 test 失败
    • 2 : Test 执行被用户打断
    • 3 : 执行测试时发生内部错误
    • 4 : pytest 命令用法错误
    • 5 : 未采集到 tests
  3. pytest –version
  4. 失败后停止
    • 第一次 pytest -x
    • 第 n 次 : pytest --maxfail=2
  5. 指定某个测试用例
    • 显式指定函数名，通过 :: 标记(一次只能指定一个测试函数)。pytest test_no_mark.py::test_func1
    • 使用模糊匹配， 使用 -k 选项标识。(可以批量操作，但需要所有测试的函数名包含相同的模式) pytest -k func1 test.py
    • 使用 pytest.mark 在函数上进行标记,测试时使用 -m 选择标记的测试函数
      • 一个函数可以打多个标记，多个函数也可以打相同的标记
      • 运行测试时使用 -m 选项可以加上逻辑
  6. Modifying Python traceback printing
  7. -r : 详细总结报告
    • f : failed
    • s : skipped
  8. 使用 debugger 调试失败的 test
    • pytest --pdb
  9. 在开始时调用 PDB ： pytest --trace
  10. breakpoint
    • ###TODO
  11. 持续时间
    • 获取执行时间超过1s的10个test : pytest --durations=10 --durations-min=1.0
  12. 使用 faulthandler 模块获取运行中进程的 traceback 信息
  13. python -m pytest [...]
    • 会将当前目录也添加到sys.path

3. 异常

  1. 断言

    • assert a == b

  2. 捕获异常

    # test_raises.py
def test_raises():
with pytest.raises(TypeError) as e:
connect('localhost', '6379')
exec_msg = e.value.args[0]
assert exec_msg == 'port type must be int'

  3. 标记函数

    • 显式指定函数名，通过 :: 标记(一次只能指定一个测试函数)。pytest test_no_mark.py::test_func1

    • 使用模糊匹配， 使用 -k 选项标识。(可以批量操作，但需要所有测试的函数名包含相同的模式) pytest -k func1 test.py

    • 使用 pytest.mark 在函数上进行标记,测试时使用 -m 选择标记的测试函数

      • 一个函数可以打多个标记，多个函数也可以打相同的标记
      • 运行测试时使用 -m 选项可以加上逻辑
      @pytest.mark.finished
def test_func1():
    assert 1 == 1

@pytest.mark.unfinished
def test_func2():
    assert 1 != 1

# $ pytest -m finished test.py

  4. 跳过测试

    • 使用 s 表示测试被跳过
    @pytest.mark.skip(reason='out-of-date api')
def test_connect():
    pass
    • 使用 pytest.mark.skipif 指定被忽略的条件
    @pytest.mark.skipif(conn.__version__ < '0.2.0',
reason='not supported until v0.2.0')
def test_api():
pass

  5. 预见的错误

    • 使用 xfail 实现预见错误功能

  6. 参数化

    • 每组参数都独立执行一次测试 pytest.mark.parametrize(argnames, argvalues)
    • 使用 pytest.param 的 id 参数进行自定义

  7. 固件 : fixture()

    • pytest 在执行测试函数前(后)加载运行他们
    • 常见使用为数据库的初始链接和最后关闭操作
    • Pytest一次只会缓存一个fixture实例
    • 较高范围的fixture(例如session会话级)较低范围的fixture(例如function函数级或class类级优先执行
    # test_postcode.py
@pytest.fixture()
def postcode():
    return '010'

def test_postcode(postcode):
    assert postcode == '010'
    • 使用 conftest.py 集中管理固件

  8. 预处理和后处理

    • 使用 yeild 关键词
      • yield 之前代码属于预处理，会在测试前执行
      • yield 之后代码将在测试完成后执行
      • 执行时使用 -s 防止消息被吞
      • 更详细: –setup-show

  9. 作用域 : scope 参数

    • function 函数级
    • class 类级别
    • module 模块级
    • session 会话级
    @pytest.fixture(scope='function')
def func_scope():
    pass

  10. 自动执行 : autouse 参数

    • 两个自动计时固件，一个用于统计每个函数运行时间，一个用于计算总耗时
    # test_autouse.py
DATE_FORMAT = '%Y-%m-%d %H:%M:%S'

@pytest.fixture(scope='session', autouse=True)
def timer_session_scope():
    start = time.time()
    print('\nstart: {}'.format(time.strftime(DATE_FORMAT, time.localtime(start))))

    yield

    finished = time.time()
    print('finished: {}'.format(time.strftime(DATE_FORMAT, time.localtime(finished)))
    print('Total time cost: {:.3f}s'.format(finished - start))


@pytest.fixture(autouse=True)
    def timer_function_scope():
        start = time.time()
        yield
        print(' Time cost: {:.3f}s'.format(time.time() - start))
