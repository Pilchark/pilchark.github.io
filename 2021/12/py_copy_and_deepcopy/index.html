浅拷贝和深拷贝

  2021-12-06   Python   python 字数统计: 660字 阅读时长: 2分
  • 参考文章 :
    • https://www.cnblogs.com/semon-code/p/8231041.html

对象赋值

  1. 对象赋值不会复制对象，它只会复制一个对象引用，不会开辟新的内存空间.
test = ["test", 18, ["python","java"]]
copytest = test
print(id(test))     # 4559314816
print(id(copytest)) # 4559314816
# 将test赋值给copytest后，test和copytest的id值相同，test和copytest指向的是同一个list.当test的值改变时，copytest的值也相应改变

拷贝

  • 浅拷贝(copy.copy(x))和深拷贝(copy.deepcopy(x))的区别只与复合对象有关系（复合对象就是对象中包含其他对象，例如list和其他实例类）
    • 浅拷贝构造了一个新的复合对象，然后（尽可能）把原来的对象引用直接插入到新对象中
    • 深拷贝构造了一个新的复合对象，然后递归的将原对象中的内容进行复制，并插入新对象

  1. 浅拷贝

    • 浅拷贝会为copytest开辟新的内存空间，但copytest中的对象仍然是test中的对象引用，不会对其中的对象进行复制
    • 对浅拷贝后的复合对象进行修改会对原对象产生影响
    import copy
test = ["test", 18, ["python","java"]]
copytest = copy.copy(test)
print(id(test))                 # 4484571008
print(id(copytest))             # 4484611320
print [id(x) for x in test]     # [4484554208,  140582137648080, 4484569424]
print [id(x) for x in copytest] # [4484554208,  140582137648080, 4484569424]

  2. 字典的浅拷贝可以使用dict.copy()，list的浅拷贝可以使用slice截取整个list，例如：copied_list = original_list[:]

  3. 深拷贝

    • 深拷贝会为copytest开辟新的内存空间，并将test中的复合对象进行拷贝，并将其引用赋值给新copytest
    import copy
test = ["test", 18, ["python","java"]]
copytest = copy.deepcopy(test)
print(id(test))                 # 4476972928
print(id(copytest))             # 4477013240
print [id(x) for x in test]     # [4476956224,  140706592084752, 4476971344]
print [id(x) for x in copytest] # [4476956224,  140706592084752, 4477013312]

  4. 深拷贝存在两个问题：一个是递归拷贝可能导致递归循环，另一个是深拷贝会拷贝所有的对象，而有些可以复用的对象也会被重复拷贝，导致拷贝过多的情况
