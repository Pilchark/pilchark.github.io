python 项目的配置文件

  2022-08-10   Python   python 字数统计: 3191字 阅读时长: 7分
  • 参考文章 :
    • https://hackersandslackers.com/simplify-your-python-projects-configuration/

我们可以使用许多文件类型来存储和访问整个项目中的基本变量。文件类型（如 ini、yaml 等）都具有在结构化（或非结构化）层次结构中存储信息的独特方式。根据项目的性质，这些文件结构中的每一个都可以很好地为您服务或妨碍您。我们将研究所有这些选项的优点，并使用适当的Python库解析这些配置。

Meet The Contenders

现代软件中格式化配置文件的方法很多。我们将介绍一些用于处理项目配置的最常见文件格式（ini，toml，yaml，json，.env）以及解析它们的Python库。

Configure from .ini

ini 文件可能是我们可用的最直接的配置文件。ini 文件非常适合较小的项目，主要是因为这些文件仅支持 1 级深度的层次结构。ini 文件本质上是平面文件，但变量可以属于组。以下示例演示了共享公共主题的变量如何属于公共标题，例如 [DATABASE] 或 [LOGS]：

# config.ini
[APP]
ENVIRONMENT = development
DEBUG = False

[DATABASE]
USERNAME = root
PASSWORD = p@ssw0rd
HOST = 127.0.0.1
PORT = 5432
DB = my_database

[LOGS]
ERRORS = logs/errors.log
INFO = data/info.log

[FILES]
STATIC_FOLDER = static
TEMPLATES_FOLDER = templates

这种结构无疑使人类更容易理解，但这种结构的实用性超出了美学。让我们用Python的configparser 库来解析这个文件，看看到底发生了什么。我们首先将 test.ini 的内容保存到一个名为 config 的变量中：

"""Load configuration from .ini file."""
import configparser

# Read local file `config.ini`.
config = configparser.ConfigParser()
config.read('settings/config.ini')

在ini文件上调用 read() 不仅仅是存储普通数据;我们的config变量现在是一个独特的数据结构，允许我们使用各种方法将值读取和写入我们的配置。尝试运行 print(config) 来亲眼看看：

<configparser.ConfigParser object at 0x10e58c390>

配置文件的存在是为了提取值的简单目的。configparser 允许我们以多种方式执行此操作。以下每一行都返回值 127.0.0.1：

"""Load configuration from .ini file."""
import configparser

# Read local `config.ini` file.
config = configparser.ConfigParser()
config.read('settings/config.ini')

# Get values from our .ini file
config.get('DATABASE', 'HOST')
config['DATABASE']['HOST']

getboolean('APP'，'DEBUG') 将正确返回布尔值False，而不是读取False的字符串，这显然对我们的应用程序是有问题的。如果我们的值DEBUG被设置为布尔值以外的值，config。getboolean()将抛出一个错误.configparser还有一系列其他类型检查方法，如’getint()’、‘getfloat()‘等等。

configparser的功能还不止于此。我们可以详细介绍该库编写新配置值、检查键的存在等等的能力，但我们不能这样做。

使用 Toml 进行配置

乍一看，TOML 文件似乎与 ini 文件有 一些 语法相似之处，但支持更广泛的数据类型以及值本身之间的关系。 TOML 文件还迫使我们预先更明确地了解数据结构，而不是像 configparser 那样在之后*确定它们。

在 Python 中解析 TOML 文件是由一个名为 toml 的库处理的，在我们去那里之前，让我们看看 TOML 是关于什么的。

TOML Variable Types

TOML 文件以类似于 ini 文件的方式通过键/值对定义变量。 这些对被称为键。然而，与 ini 文件不同的是，TOML 期望键的值存储为它们打算用作的数据类型。旨在被解析为字符串的变量必须存储为引号中的值，而布尔值必须存储为原始“true”或“false”值。这消除了我们配置的许多歧义：我们不需要 TOML 文件的方法，例如 getboolean() 。

TOML 文件可以支持令人印象深刻的变量类型目录。一些更令人印象深刻的 TOML 变量类型包括DateTime、local time、arrays、floats，甚至hexadecimal values：

[project]
name = "Faceback"
description = "Powerful AI which renders the back of somebody's head, based on their face."
version = "1.0.0"
updated = 1979-05-27T07:32:00Z
author = "Todd Birchard"

...

config.toml

TOML File Structures

TOML 文件中括号内的部分称为“表格”。 Keys 可以存在于表的内部或外部，正如我们在下面的示例中看到的那样。您会注意到，这不是 TOML 文件的仅有的两个元素：

# Keys
title = "My TOML Config"

# Tables
[project]
name = "Faceback"
description = "Powerful AI which renders the back of somebody's head, based on their face."
version = "1.0.0"
updated = 1979-05-27T07:32:00Z
author = "Todd Birchard"

[database]
host = "127.0.0.1"
password = "p@ssw0rd"
port = 5432
name = "my_database"
connection_max = 5000
enabled = true

# Nested `tables`
[environments]
  [environments.dev]
    ip = "10.0.0.1"
    dc = "eqdc10"
  [environments.staging]
    ip = "10.0.0.2"
    dc = "eqdc10"
  [environments.production]
    ip = "10.0.0.3"
    dc = "eqdc10"

# Array of Tables
[[testers]]
id = 1
username = "JohnCena"
password = "YouCantSeeMe69"

[[testers]]
id = 3
username = "TheRock"
password = "CantCook123"

config.toml

TOML supports the concept of "nested tables," as seen in the [environments] table, preceded by multiple sub-tables. Using dot-notation enables us to create associations of tables, which imply they’re different instances of the same element.

Equally impressive is the concept of "arrays of tables," which what’s happening with [[testers]]. Tables in double-brackets are automatically added to an array, where each item in the array is a table with the same name. The best way to visualize what’s happening here is with the JSON equivalent:

{
  "testers": [
    { "id": 1, "username": "JohnCena", "password": "YouCantSeeMe69" },
    { "id": 2, "username": "TheRock", "password": "CantCook123" }
  ]
}

Parsing TOML

Enough about TOML as a standard, let’s get our data:

"""Load configuration from .toml file."""
import toml

# Read local `config.toml` file.
config = toml.load('settings/config.toml')
print(config)

Python dict parsed from config.toml

Grabbing values from config is as easy as working with any dictionary:

"""Load configuration from .toml file."""
import toml

# Read local `config.toml` file.
config = toml.load('settings/config.toml')
print(config)

# Retrieving a dictionary of values
config['project']
config.get('project')

# Retrieving a value
config['project']['author']
config.get('project').get('author')

Configure from .yaml

YAML 文件格式已成为人们最喜欢的配置文件格式，大概是因为它们易于阅读。熟悉 YAML 规范的人会告诉您，YAML 远不是一种优雅的文件格式，但这并没有阻止任何人去使用。

在 Python 中解析 YAML

我推荐 Python Confuse 库（肯定会引起贵公司信息安全团队的注意的包名称）。

Confuse 允许我们以与我们与 JSON 几乎相同的方式与 YAML 文件进行交互。例外情况是 confuse 库需要在键上指定 .get() 以提取其值，如下所示：

"""Load configuration from .yaml file."""
import confuse

config = confuse.Configuration('MyApp', __name__)
runtime = config['AWS']['Lambda']['Runtime'].get()
print(runtime)

yaml_config.py

.get() 可以接受数据类型值，例如 int。 这样做可以确保我们获得的值实际上是我们期望的模式，这是一个很好的功能。

验证者

Confuse 的文档 详细介绍了我们从 YAML 文件中提取的值的其他验证方法。 as_filename()、as_number() 和 as_str_seq() 等方法基本上可以满足您的期望。

命令行配置

Confuse 还进入了构建 CLI 的领域，允许我们使用我们的 YAML 文件来通知可以传递给 CLI 的参数及其潜在值：

config = confuse.Configuration('myapp')
parser = argparse.ArgumentParser()

parser.add_argument('--foo', help='a parameter')

args = parser.parse_args()
config.set_args(args)

print(config['foo'].get())

Configure from .env

环境变量是将敏感信息排除在项目代码库之外的好方法。我们可以通过多种不同的方式存储环境变量，其中最简单的是通过命令行：

$ export MY_VARIABLE=AAAAtpl%2Bkvro%2BoQ9wRg77VUEpQv%2F

export MY_VARIABLE

以这种方式存储的变量只会在您当前的终端会话打开时持续存在，因此这在测试之外对我们没有太大帮助。如果我们希望 MY_VARIABLE 持续存在，我们可以将上面的 export 行添加到我们的 .bash_profile（或等效文件）以确保 MY_VARIABLE 将始终存在于系统范围内。

项目特定的变量更适合位于我们项目目录中的 .env 文件。看在上帝的份上，不要将这些文件提交给 GITHUB。

假设我们有一个 .env 文件，其中包含与项目相关的变量，如下所示：

FLASK_ENV=development
FLASK_APP=wsgi.py
COMPRESSOR_DEBUG=True
STATIC_FOLDER=static
TEMPLATES_FOLDER=templates

.env

We can now extract these values in Python using the built-in os.environ:

"""App configuration."""
from os import environ, path
from dotenv import load_dotenv

# Find .env file
basedir = path.abspath(path.dirname(__file__))
load_dotenv(path.join(basedir, '.env'))

# General Config
SECRET_KEY = environ.get('SECRET_KEY')
FLASK_APP = environ.get('FLASK_APP')
FLASK_ENV = environ.get('FLASK_ENV')
