面向对象编程的 SOLID 原则 (python 语言)

  2022-12-12   Python OOP   python oop 字数统计: 1028字 阅读时长: 3分

https://www.freecodecamp.org/chinese/news/solid-principles/ https://towardsdatascience.com/solid-coding-in-python-1281392a6a94 https://medium.com/carbon-consulting/solid-principles-with-python-245e45f9b1f8

背景

SOLID 原则首先由著名的计算机科学家 Robert C·Martin （著名的Bob大叔）由 2000 年在他的论文中提出。但是 SOLID 缩略词是稍晚由 Michael Feathers 先使用的。
Bob大叔也是畅销书《代码整洁之道》和《架构整洁之道》的作者，也是 “Agile Alliance” 的成员。

SOLID 是以下是原则的缩写： - S 单一职责原则
- O 开闭原则
- L 里氏替换原则
- I 接口隔离原则
- D 依赖倒置原则

单一职责原则: The Single-Responsibility Principle (SRP)

  • 一个 class 应该只做一件事，一个 class 应该只有一个变化的原因

  1. 例子 : 计算一组数的平均数和中位数

    import numpy as np
def math_operations(list_):
    # Compute Average
    print(f"the mean is {np.mean(list_)}")
    # Compute Max
    print(f"the max is {np.max(list_)}") 

math_operations(list_ = [1,2,3,4,5])
# the mean is 3.0
# the max is 5

  2. 将函数拆分原子化，然后创建一个 class 或 function，分别调用函数。

    def get_mean(list_):
    '''Compute Mean'''
    print(f"the mean is {np.mean(list_)}") 

def get_max(list_):
    '''Compute Max'''
    print(f"the max is {np.max(list_)}") 

def main(list_): 
    # Compute Average
    get_mean(list_)
    # Compute Max
    get_max(list_)

main([1,2,3,4,5])
# the mean is 3.0
# the max is 5

  3. 好处

    • 方便定位错误代码
    • 代码可复用
    • 方便为每个函数创建测试用例

开闭原则(The Open–closed principle (OCP))

  • 开闭原则要求“class 应该对扩展开放对修改关闭”。
  1. 使用 __subclasses__() 调用所有继承自抽象类的子类的方法。
import numpy as np
from abc import ABC, abstractmethod

class Operations(ABC):
    '''Operations'''
    @abstractmethod
    def operation():
        pass

class Mean(Operations):
    '''Compute Max'''
    def operation(list_):
        print(f"The mean is {np.mean(list_)}") 

class Max(Operations):
    '''Compute Max'''
    def operation(list_):
        print(f"The max is {np.max(list_)}") 

class Main:
    '''Main'''
    @abstractmethod
    def get_operations(list_):
        # __subclasses__ will found all classes inheriting from Operations
        for operation in Operations.__subclasses__():
            operation.operation(list_)


if __name__ == "__main__":
    Main.get_operations([1,2,3,4,5])
# The mean is 3.0
# The max is 5

里氏替换原则: The Liskov substitution principle (LSP)

  • 子类应该能替换为它的基类

  1. 如果在子类中重定义了基类中的函数，则这两个函数应具有相同的行为。但并不意味它们必须完全相等，而是在用户给定相同的输入后，显示相同类型的结果。

  2. 例子: 求长方形面积类

from abc import ABC
class Rectangle(abc):
    def __init__(self, width, height) -> None:
        self.width = width
        self.height = height
    def get_width(self):
        return self.width
    def set_width(self, width):
        self.width = width
    def get_height(self):
        return self.height
    def set_height(self, height):
        self.height = height
    def get_area(self):
        return self.width * self.height

现在准备创建另一个 Squares class,求正方形面积

class Square(Rectangle):
    def __init__(self, size) -> None:
        self.width = self.height = self.size = size
    def set_width(self, width):
        super().set_height(width)
        super().set_width(width)
    def set_height(self, height):
        super().set_height(height)
        super().set_width(height)

我们不希望任何客户端（在他们的代码里使用我们的 class）违背了正方形的特性将宽高改成不相等。 因此我们重载了 setter 使宽和高任何一个改变时都会同时改变宽高。这样一来，我们就违背了里氏替换原则。

接口隔离原则: The Interface Segregation Principle (ISP)

  • 隔离意味着保持独立，接口隔离原则是关于接口的独立。
  1. 例子
import numpy as np
from abc import ABC, abstractmethod

class Mammals(ABC):
    @abstractmethod
    def swim() -> bool:
        print("Can Swim") 

    @abstractmethod
    def walk() -> bool:
        print("Can Walk") 

class Human(Mammals):
    def swim():
        return print("Humans can swim") 

    def walk():
        return print("Humans can walk") 

class Whale(Mammals):
    def swim():
        return print("Whales can swim")

人类和鲸鱼类继承自哺乳动物类

Human.swim()
Human.walk()

Whale.swim()
Whale.walk()

# Humans can swim
# Humans can walk
# Whales can swim
# Can Walk

子类鲸鱼理论上不应该但仍然可调用 walk 方法。 应该创建更多的用户接口，避免使用一个通用接口。

from abc import ABC, abstractmethod

class Walker(ABC):
  @abstractmethod
  def walk() -> bool:
    return print("Can Walk") 

class Swimmer(ABC):
  @abstractmethod
  def swim() -> bool:
    return print("Can Swim") 

class Human(Walker, Swimmer):
  def walk():
    return print("Humans can walk") 
  def swim():
    return print("Humans can swim") 

class Whale(Swimmer):
  def swim():
    return print("Whales can swim") 

if __name__ == "__main__":
  Human.walk()
  Human.swim()

  Whale.swim()
  Whale.walk()

# Humans can walk
# Humans can swim
# Whales can swim

依赖倒置原则: The Dependency Inversion Principle (DIP)

class 应该依赖接口和抽象类而不是具体的类和函数。
