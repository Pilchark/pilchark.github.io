了解 python 虚拟环境

  2022-04-25   Python   python 字数统计: 324字 阅读时长: 1分
  • 参考文章 : https://realpython.com/python-virtual-environments-a-primer/

为什么使用 venv

  1. 安装同库不同版本时避免后者覆盖前者
  2. 利于管理不同项目的依赖
  3. 远程部署环境时可自由安装库

venv 是什么

  1. Include/ : 初始化为空文件夹, 包含 C 头文件
  2. Lib/ : python 库的安装位置
  3. Scripts/ : 进入 python 虚拟环境的可执行文件
  4. pyvenv.cfg : 配置文件

venv 是怎样工作的

  1. 解释器查找 pyvenv.vfg 文件, sys.prefix 指向包含 pyvenv.vfg 的文件

    import sys
sys.prefix
sys.base_prefix

  2. venv 会链接到标准 python 环境中的库

    import sys
from pprint import pp
pp(sys.path)

自定义 venv

  1. 改变 venv 文件夹名
    python -m venv your-fancy-name
  2. 同时改变命令行提示名且不影响
    python -m venv venv --prompt="dev-env"
  3. 创建一个新的 venv,同时移除所有旧的 pip 库
    python -m venv venv --clear
  4. 一次创建多个 venv(传入多个路径)
    python -m venv venv /my/path/copy(2个)
  5. 创建 venv 后自动升级 pip
    python -m venv venv --upgrade-deps
  6. 使 venv 中包含系统环境中的 pip 库
    python -m venv venv --system-site-packages

延伸

  1. Virtualenv : venv 的超集，扩展性更高
  2. Conda : 提供 python 和其他语言的包，依赖。

备份依赖

  1. python -m pip freeze > requirements.txt
